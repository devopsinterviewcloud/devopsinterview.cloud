Date: 2026-06-14
Scope: READ-ONLY Next.js/DevOps build audit. No files modified.
Verification: `npx tsc --noEmit` (run, FAILS), `npx jest` (run, FAILS — no tests), file reads, grep dependency/import analysis.

## Summary

- Critical: 3
- High: 4

Most serious: **`@supabase/supabase-js` is declared in package.json but is NOT in package-lock.json and NOT installed.** `src/lib/storage.ts` imports it, so `tsc --noEmit` fails and a clean Vercel build (`npm ci` + `next build` with `typescript.ignoreBuildErrors: false`) will fail. tsc: FAIL (1 error). jest: FAIL (0 tests found, exits 1).

## What passes / positives

- TypeScript config is strict (`strict: true`, `noEmit`, isolatedModules); no `// @ts-ignore` or stray `any` hotspots found in the payment libs (paypal.ts/razorpay.ts use proper typed returns and `unknown`).
- Payment code is well-built: REST-based (no SDK bloat), webhook treated as source of truth, HMAC verification uses `crypto.timingSafeEqual` (download-token.ts:33, razorpay.ts:55-60), PayPal webhook verified via PayPal's verify-signature API.
- `removeConsole` in production (excludes error/warn), `poweredByHeader:false`, `productionBrowserSourceMaps:false` — good hardening.
- Accessibility quick pass OK: `<html lang="en">` (layout.tsx:76), CookieConsent has `aria-label`/`aria-live`, breadcrumb nav labeled, all `<button>`s carry text (no unlabeled icon-only buttons found).
- Prisma client IS generated; provider postgresql with `env("DATABASE_URL")`.
- Webhooks correctly excluded from rate limiting (so gateways can retry).
